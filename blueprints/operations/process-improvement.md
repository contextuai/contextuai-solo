# Process Improvement (Six Sigma DMAIC)

A structured process improvement framework based on the Six Sigma DMAIC methodology. This blueprint guides teams through five phases of disciplined problem-solving to reduce waste, eliminate defects, and improve operational efficiency using data-driven analysis rather than assumptions.

## Objective

Identify, analyze, and resolve a specific process inefficiency using the DMAIC framework, producing measurable improvements in quality, speed, or cost along with controls to sustain the gains over time.

## Steps

### 1. Define the Problem
Clearly articulate the process problem you are trying to solve. Write a problem statement that specifies what is going wrong, where it occurs, when it started, and how significant the impact is in measurable terms (cost, time, error rate, customer complaints). Define the project scope by establishing clear boundaries: what is included and what is out of scope. Identify the process owner, key stakeholders, and the team members who will work on the improvement. Set a target improvement goal and a timeline for achieving it.

### 2. Map and Measure the Current State
Document the existing process in detail using a process map or flowchart. Walk through the process step by step with the people who actually perform it, capturing inputs, outputs, decision points, handoffs, and wait times. Collect baseline data on the key metrics identified in the Define phase. Measure cycle time, throughput, defect rate, rework percentage, and any other relevant performance indicators. Ensure your data collection is reliable by verifying measurement systems and gathering enough data points to be statistically meaningful.

### 3. Analyze Root Causes
Use the baseline data and process map to identify where problems originate. Start with pattern analysis: when do defects occur most frequently, which process steps have the longest delays, and where do the most handoff errors happen? Apply root cause analysis techniques such as the 5 Whys, fishbone (Ishikawa) diagrams, and Pareto analysis to move beyond symptoms to underlying causes. Validate suspected root causes with data. For each confirmed root cause, quantify its contribution to the overall problem so you can prioritize which causes to address first.

### 4. Develop and Implement Improvements
Generate multiple potential solutions for each validated root cause. Evaluate solutions against criteria including effectiveness, implementation cost, time to implement, and risk of unintended consequences. Select the best solutions and create a detailed implementation plan with specific actions, owners, and deadlines. Pilot improvements on a small scale first to validate they work as expected before rolling out broadly. Collect data during the pilot to compare against the baseline measurements and confirm the improvement is real and significant.

### 5. Establish Controls and Sustain Gains
Design control mechanisms that ensure the process stays improved after the project team moves on. Create standard operating procedures that document the new process. Implement monitoring dashboards or regular audits that track the key metrics on an ongoing basis. Define trigger points that signal when the process is drifting back toward its old performance level and specify what corrective actions should be taken. Train all process participants on the new procedures and the rationale behind the changes. Schedule a follow-up review 30, 60, and 90 days after implementation to verify that improvements are holding.

## Expected Output

A DMAIC project report containing the problem statement and scope, a current-state process map with baseline metrics, root cause analysis with supporting data, implemented solutions with pilot results, updated process documentation, a control plan with monitoring dashboards, and a summary of measured improvements versus the original target.

## Recommended Agents

- **COO** — Sponsors the improvement initiative, removes organizational barriers, and ensures changes are adopted across the operation.
- **Data Analyst** — Collects and analyzes baseline data, validates root causes statistically, and builds monitoring dashboards for the control phase.
- **Product Manager** — Helps define the problem scope, evaluates solution trade-offs, and ensures improvements align with customer-facing quality standards.

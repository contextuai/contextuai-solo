# Video 8: Workspace — Run Projects

**Duration:** 3 minutes
**Goal:** Show the workspace project lifecycle — create, execute, review discussion and compiled output.

---

## Opening (0:00 - 0:10)

**Voiceover:**
> "Workspace is where you run multi-agent projects and get structured deliverables. Think of it as your project board — create a brief, assign agents, run it, and review the output."

---

## Scene 1: Creating a Project (0:10 - 0:55)

**On screen:** Workspace page → "New Project"

**Steps:**
1. Show the empty state: "Start your first project"
2. Click **New Project** — wizard opens

**Step 1: Project Details**
3. Enter name: "Q2 Market Analysis for SaaS Vertical"
4. Enter description: "Research the B2B SaaS market landscape, identify top 5 competitors, analyze pricing strategies, and recommend positioning opportunities."
5. Show the **"Use Blueprint"** button — click it, select "Market Research" blueprint
6. Select model: "Auto"

**Step 2: Agent Selection**
7. Search "competitive" → check **Competitive Intelligence Analyst**
8. Search "data" → check **Data Analyst**
9. Search "strategist" → check **CEO Strategic Advisor**
10. Show the count badge: "3 selected"

**Step 3: Review**
11. Show the summary: project name, description, 3 agents, model
12. Click **Create**

**Voiceover:**
> "Name your project and describe what you need. Use a blueprint for a structured starting point. Then pick your agents — I'm choosing a Competitive Intelligence Analyst, a Data Analyst, and a CEO Strategic Advisor. Three specialists for one project. Review and create."

---

## Scene 2: Running the Project (0:55 - 1:30)

**On screen:** Project card in the list → Execute

**Steps:**
1. Show the project card with "Draft" status
2. Click the project card to open it
3. Click **Execute**
4. Show status changing: Draft → Running
5. Wait for completion (or use a pre-recorded clip of a completed project)
6. Status changes to **Completed** with green checkmark

**Voiceover:**
> "Your project starts as a Draft. Click Execute and the agents get to work. Each one contributes their expertise in sequence. Depending on the model and number of agents, this takes a few seconds to a few minutes."

---

## Scene 3: Reviewing Results (1:30 - 2:25)

**On screen:** Project results page — two tabs

**Discussion Tab:**
1. Show the threaded view — each agent's contribution labeled with their name
2. Scroll through: Competitive Intelligence Analyst's research, Data Analyst's findings, CEO Strategic Advisor's recommendations
3. Point out how each agent built on the previous one's work

**Compiled Output Tab:**
4. Switch to Compiled Output
5. Show the final, consolidated deliverable
6. Click the **copy button** to grab the full text

**Voiceover:**
> "Results come in two views. The Discussion tab shows each agent's individual contribution — like reading the meeting notes. You can see how the analyst's research informed the data analysis, which shaped the strategic recommendations. The Compiled Output tab gives you the final deliverable — polished and ready to use. Copy it with one click."

---

## Scene 4: Managing Projects (2:25 - 2:50)

**On screen:** Project list with multiple projects

**Steps:**
1. Show multiple project cards with different statuses (Draft, Completed, Running)
2. Click status filter pills: "Completed" → show only completed
3. Click "All" to reset
4. Show the refresh button

**Voiceover:**
> "Your workspace grows into a portfolio of analyses, reports, and deliverables. Filter by status to find what you need. Every project is saved — revisit old work anytime."

---

## Closing (2:50 - 3:00)

**Voiceover:**
> "Workspace gives you a structured way to tackle complex tasks with multiple experts. And the output doesn't have to stop here — you can feed workspace results into a crew for further processing and publishing. We'll show that workflow in the advanced series."

**End card:** "Next: Connections — Social Publishing"
